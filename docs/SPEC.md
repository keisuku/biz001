# Pivot Bench 仕様書

> 元の凍結仕様はチャット上で策定されリポジトリに含まれていなかったため、本書は M0 実装・PR #1 の記述・README から読み取れる仕様を書き起こし、以後の基準とするもの。

## 目的

日本の法人保険代理店向けに、SalesforceレポートのCSV/xlsxをオフラインで正規化し、バケットマスタで商材3階層分類を派生させ、FINOS Perspectiveで高速ピボット表示・比較する。

## 配布形態・制約

- 成果物は **単一HTMLファイル** `dist/pivot-bench.html`(約7.4MB)。
- 実行時に **ネットワークアクセスを一切行わない**(CDN・fetch・外部フォント禁止)。Perspective/SheetJSのJS・WASMはビルド時にすべて埋め込む。
- **localStorageを使わない**。マスタ・ビュー設定の永続化はJSONファイルの書き出し/読み込みで行う。
- `file://` で直接開いて動作すること(Web Worker不可の環境ではPerspectiveがシングルスレッドにフォールバック)。
- UIは日本語。ダークテーマ(CSSトークンは `src/styles.css` の `:root` で凍結)。

## 入力

- `.csv` / `.xlsx`(先頭シートを使用)。1行目がヘッダ。
- CSVの文字コードは **UTF-8 / Shift_JIS を自動判定**(UTF-8でデコードして置換文字が出たらShift_JISで再デコード)。

## 正規化ルール

列の型は先頭200行のサンプルから推定(非null値の80%超が数値解釈可能なら数値列)。

| 入力 | 変換 |
|---|---|
| 全角英数字 (Ａ-Ｚ ａ-ｚ ０-９) | 半角へ |
| 全角スペース | 半角スペースへ、前後トリム |
| 空文字・`-`・`ー` | `null` |
| `△123` / `(123)` | 負数 `-123` |
| `¥` `￥` `\`(SJISの0x5C円記号)・カンマ | 除去して数値化 |

## バケットマスタ

独立したJSON資産。構造:

```json
{
  "meta": {"name": "商材分類 v1", "updated": "YYYY-MM-DD", "note": ""},
  "target_column_candidates": ["商品コード", "商品CD", "PRODUCT_CODE"],
  "levels": ["大分類", "中分類", "小分類"],
  "rules": [
    {"match": {"type": "exact",  "value": "F001"},      "assign": ["火災", "企業財産", "オールリスク"]},
    {"match": {"type": "prefix", "value": "F"},          "assign": ["火災", "企業財産", "その他"]},
    {"match": {"type": "regex",  "value": "^A[0-9]{3}$"}, "assign": ["自動車", "フリート", "その他"]},
    {"match": {"type": "range",  "min": 1000, "max": 1999}, "assign": ["新種", "賠償", "その他"]}
  ],
  "unmatched": ["未分類", "未分類", "未分類"]
}
```

- 対象列は `target_column_candidates` を**先頭から**照合し、最初にデータへ存在した列を使う。見つからない場合は警告を出し全行 `unmatched` になる。
- ルールは**上から順に評価し、最初に一致したもので確定**(first-match-wins)。
- match type: `exact`(正規化後の完全一致)/ `prefix`(前方一致)/ `regex` / `range`(数値範囲、両端含む)。
- 一致しなかった行数を「未分類」ゲージに表示(1件以上で警告色)。
- 同名の組込み標準マスタをアプリに内蔵(初期値)。

## ピボット表示

- ピボットエンジンは自作せず `@finos/perspective` + `perspective-viewer` + `datagrid` プラグインを使用。
- 各面は独立に操作可能(Perspective標準UI)で、プリセットバーからの一括操作を基本とする。

## 複数レポート (M1)

- レポートは**最大3本**まで読込可能。読込済みレポートはチップ(A/B/C・行数・除去✕)で表示。
- **並置モード**: 面ごとに表示レポートを割当(面ヘッダのセレクタ)。同一レポートは1つのPerspectiveテーブルを共有。
- **結合モード**: 全レポートを列の和集合でunion(欠損はnull)し、出所を示す**「レポート」列**を付与して1テーブル化。切替時、列分割が未選択なら自動で「レポート」を設定(レポート横並び比較になる)。
- バケットマスタは各レポートに個別適用。未分類ゲージは全レポート合計。

## プリセットバー (M1) — Salesforceレポート上位互換の核心

Perspectiveの設定ドロワーを開かずに、その場で集計を組む(Tadの1パネルUXを踏襲):
- **行キー最大4個**(大中小+第4階層。選択順=階層順) × **列分割1個**(split_by) × **値2個**(列+集計 sum/count/avg/min/max)
- 変更は即適用。適用先は「全面」チェックで全ペイン一括/オフでアクティブペインのみ(面別比較)
- データに存在しない列は面ごとに自動スキップ。値が未選択なら最初の数値列をsumで自動選択

## 永続化(JSON入出力)

- **マスタ**: 「マスタを開く」(JSONファイル選択、構造検証あり)/「マスタ書き出し」(現在のマスタをダウンロード)。
- **ビュー設定 v2**: 「ビュー書き出し」で `{"kind": "pivot-bench-views", "version": 2, "combine": bool, "assignments": [..], "preset": {..}, "views": [viewer.save()×3]}` をダウンロード。「ビュー読込」で結合状態・面割当・プリセット・各面ビューまで復元。v1(viewsのみ)も読込可。

## 操作

- ファイルは「開く」「＋レポート追加」ボタン、Cmd/Ctrl+O、または**ページへのドラッグ&ドロップ**で読込(複数選択可)。CSV/xlsxはレポート、JSONはマスタとして扱う。
- ショートカット: Cmd/Ctrl+O 開く / Cmd/Ctrl+Enter 適用 / 1・2・3 アクティブペイン切替 / Esc ダイアログ閉じる。
- アクティブペインは枠がアンバーに発光。ペインクリックでも切替。

## 検証基準

- `testdata/pivot-bench-50000.xlsx`(50列×50,000行、`npm run generate:testdata` で生成)を読み込み、3面のピボットが表示されること。
- 標準マスタ適用時の未分類件数が 2,942(=⌈50000/17⌉)であること。
- 2本目のレポート追加→結合モードで行数が合計と一致し、「レポート」列で列分割されること。
- ビュー設定v2の書き出し→読み込みで結合状態・面別の分割状態が再現されること。
- 実行時のhttp(s)リクエストが0件であること。

## 入力仕様(ここが肝。実物CSVの癖に耐えること)

1. **GA4のUIエクスポートCSV**(「レポート→共有→ファイルをダウンロード→CSV」):
   - 先頭に `#` で始まるメタ行(レポート名・期間)が付き、空行区切りで複数の表が入ることがある → `#`行をスキップし、最大の表を主表として採用。期間はメタ行から抽出(取れなければファイル名の `YYYYMM`、それも無ければ画面で月を指定させる)
   - 列名はプロパティの言語設定で日英どちらもあり得る(例: 「セッションのデフォルトチャネルグループ」/ "Session default channel group") → 主要列は日英両対応の辞書で解決
2. **Search ConsoleのエクスポートCSV**(「検索パフォーマンス→エクスポート」。zip内の クエリ.csv / ページ.csv / Queries.csv / Pages.csv 等):
   - ヘッダ(クエリ/Query、ページ/Top pages、クリック数/Clicks、表示回数/Impressions、CTR、掲載順位/Position)で種別を自動判別。CTRの「%」や小数の掲載順位を数値化
3. **ファイル種別の自動判別**: ドロップされた各CSVを「GA4チャネル」「GA4ランディングページ」「GSCクエリ」「GSCページ」等に分類してチップ表示(判別不能は「不明」チップにして無視。壊れないこと)
4. **複数月**: 同種CSVを複数月分ドロップすると月次の時系列として結合(Pivot Benchの結合思想)。同じ月の同種は後勝ち

## 月報の章立て(画面と書き出しHTMLで同一構成)

1. **結論サマリー** — 前月比の要点を自動生成した3行(総セッション増減、最伸長/最下落チャネル、異常件数)
2. **流入層** — チャネル別 sessions / engaged sessions / new users の表+前月比。傾向のミニ棒(CSSで可。チャートライブラリ追加禁止ではないが単一HTML・オフラインを守ること)
3. **LP層** — ランディングページ別 sessions / エンゲージ率 / コンバージョン(問い合わせ)。歩留まり列(sessions→問い合わせ率)
4. **SEO層** — クエリ別 impressions / clicks / CTR / 掲載順位、上位変動(前月比で順位が大きく動いたクエリを強調)
5. **変化層(異常)** — 各指標のrolling z-score(直近n ヶ月平均±σ)で |z|≧2 を異常として一覧。「何が異常か」だけでなく「どの次元(チャネル/LP/クエリ)が作ったか」を1行添える
6. **データの注記** — 読み込んだファイル一覧・期間・判別結果(監査可能性)

## 機能

- 書き出し: ①月報を**スタンドアロンHTML**(それ自体もネットワーク0件)でダウンロード ②Markdown版 ③**AI相談バンドル**(データ概要+主要表+「この傾向の解釈は?」定型文3種。やること/4と同じ思想、クリップボードにもコピー)
- 設定(コンバージョン扱いにするイベント名、異常検知のσ閾値、月数)はJSON書き出し/読み込みで永続化(localStorage禁止)
- ガジェット感: 起動→ドロップ→月報まで3操作以内。数字はモノスペース・tabular-nums。前月比は色(緑/赤)と▲▼で一目
- **サンプルデータ生成**: `ga4-bench/build/generate-sample.mjs` で、GA4/GSCの実エクスポート形式を忠実に模した3ヶ月分のサンプルCSV一式(メタ行・日本語列名・%表記・異常を1つ仕込む)を `testdata/ga4-sample/` に生成。実データ到着前の開発・検証はこれで行う

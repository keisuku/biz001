## 背景

Pivot Bench はSalesforceレポートをオフラインで正規化・分類・3面ピボット比較する持ち主専用の単一HTMLツール。前バージョンでプリセットバーと複数レポートの並置/結合まで完成した。今回は日常運用の摩擦を消す: マスタをJSON手編集せずに画面で直せること、よく使う集計を1クリックで呼べること、結果を他所へ持ち出せること。

## 着手前に読むファイル

- `atelier/憲法.md` — 道具づくりの8原則(持ち主ひとり専用/オフライン単一HTML/格好良さは必須要件)
- `docs/SPEC.md` — 現仕様。特にバケットマスタのJSON構造と「上から順に最初の一致」の評価
- `atelier/調査結果/ピボットBIの最高峰調べ.md` — 参考にする既存ツール(特にHueyの書き出し機能)

## 前提・制約

- 実行時ネットワーク0件・localStorage不使用(自動テストで担保済み。壊さないこと)
- 既存のダークテーマ(`src/styles.css` の `:root` の色)のトーンを維持
- ビルド: `npm install && npm run build` → `dist/pivot-bench.html`。テストデータ: `npm run generate:testdata`
- `src/app.js` の既存部品(`defaultMaster` / `applyMaster()` / `download()` / ビュー設定JSON入出力)を再利用する

## 作るもの

1. **マスタ編集画面**: 現在のマスタをモーダル内に表形式で表示(ルール行: 種別/値/割当3階層、↑↓ボタンで順序変更、行の追加/削除)。名前・更新日も編集可。「適用」で即再集計、「JSONへ書き出し」は既存 `download()` を使う
2. **ルール別ヒット数**: 編集画面内に、直近の適用でルールごとに何件当たったか・未分類が何件かを表示
3. **お気に入り集計**: 名前を付けたプリセット(行キー・列分割・値・集計の組)を複数保存し、ヘッダーのドロップダウンから1クリック適用。保存先はビュー設定JSON(localStorageは禁止)
4. **結果の書き出し**: アクティブなペインの集計結果をCSVダウンロード+クリップボードコピー(タブ区切り、Excel貼り付け用)。Perspectiveの `view.to_csv()` を使う
5. ボタンが増えてヘッダーが窮屈になったら、マスタ関連を1つのメニューに集約してよい

## やらないこと

グラフ/チャート、データベースへの乗り換え、複数マスタの同時適用、多言語対応、localStorage

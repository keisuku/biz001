# 1. Obsidianに知識の保管庫を作ってもらう

> **■使い方(3行)**
> ① ChatGPTかClaudeのチャットを開く(パソコンでClaude Codeが使えるならそちらが一番楽)
> ② 下の「----- ここから下を全部コピーして貼る -----」より下を全部貼り付ける
> ③ AIがファイルの中身を全部出してくれるので、言われたとおりフォルダに保存する(Claude Codeなら自動で作ってくれる)

でき上がるもの: パソコンの中に「アイデアや調べ物を貯めておく保管庫」フォルダ一式。Obsidianという無料アプリで開くと綺麗に見えますが、ただのフォルダとテキストなのでObsidianが無くても使えます。

----- ここから下を全部コピーして貼る -----

あなたに、私専用の「知識の保管庫」を作ってほしいです。パソコンに新しいフォルダ(名前: atelier-vault)を作り、その中に以下のフォルダとファイル一式を用意します。あなたがファイルを直接作れる環境ならそのまま作ってください。作れない環境なら、各ファイルの中身を全文出力して、私が保存できるようにしてください。

【目的】
「欲しい道具のアイデア → 世界の最高峰事例の調査 → AIへのお願い書」を全部プレーンなMarkdownファイルで一元管理し、AIがフォルダごと読み書きできるようにする。Obsidianという無料アプリで開ける形式にするが、Obsidianが無くても壊れない構造にする。

【フォルダ構成】
- 00_思いつき/ … 「〜が欲しい気がする」と思った瞬間に1行書き捨てる場所。整理不要
- 10_欲しいもの/ … 思いつきを下のひな形に沿って育てたメモ
- 20_調べたこと/ … 世界の最高峰ツールを調べた記録
- 30_お願い書/ … AIに渡す作業依頼書
- 40_資産/ … 業務で使うマスタJSON・ルール類の保管
- 50_目次/ … 領域別のまとめノート(仕事/知的生産/私生活)
- 90_保管庫/ … 終わったもの・やめたもの

【全ファイル共通ルール】
新しいノートには必ず冒頭にこの3行を付ける(AI検索の精度が大きく上がるため):

```
---
summary: (1行の要約)
status: 思いつき
date: (日付)
---
```

【10_欲しいもの/ に置くひな形(「ひな形.md」として保存)】

```
# (仮の名前)
## 思いつき(生の言葉のまま)
## はっきりさせる
- 何をする道具か:
- 週に何回使うか:
- いつ・どこで・何のデータで使うか:
- 今は何でやっていて何分かかるか:
- 完成した道具を使っている自分の情景(3行):
## いる・いらない
- 絶対いる:
- あったら嬉しい:
- 思い切っていらない:
- 見た目・手触りの希望(テンションが上がる条件):
```

【CLAUDE.md(フォルダ直下に保存。AIへの申し送り)】
内容: このフォルダは持ち主専用の知識の保管庫。ルール: (1)新規ノートには必ずsummary/status/dateの3行を付ける (2)00_思いつき/にメモを見つけたら、10_欲しいもの/のひな形に育てることを提案する (3)ゼロから何かを発明しない。必ず既存の最高峰事例を調べてから作る (4)持ち主ひとりのための道具だけを考える。汎用化の提案はしない

【最後に】
README.md をフォルダ直下に作り、各フォルダの意味とこの運用ルールを、初めて見る人にも分かる言葉で1ページにまとめてください。Obsidianを使う場合のおすすめ無料プラグイン(Dataview、Templater、QuickAdd、obsidian-git)も用途1行ずつ添えてください。

# 2. 世界の最高峰ツールを調べてもらう

> **■使い方(3行)**
> ① ChatGPT(できればDeep Research機能)かGeminiを開く
> ② 下の【調査依頼文】のうちやりたいものを1本、枠ごと全部コピーして貼る
> ③ 返ってきた調査結果を、このフォルダの「調査結果/」に(または保管庫の「20_調べたこと/」に)そのまま保存する

なぜやるか: このアトリエの鉄則は「ゼロから作らない。世界の最高峰を調べてから、良いとこ取りで作る」だからです。調べる作業はAIに任せられるので、依頼文を定型化してあります。

## 調査依頼文① ルール表を画面で編集するUIの最高峰

```
あなたは「世界の最高峰事例を調べ尽くす調査員」です。以下の需要を満たす既存の
ソフトウェア/OSS/商用製品を世界単位で調査してください。

【需要】表形式の変換ルール(if-thenのルール一覧)をブラウザ上で編集・並べ替え・
検証できるUIが欲しい。個人利用・オフライン動作優先。
【調査範囲】GitHub(スター順と直近1年更新の両方)、商用製品、Hacker News、
Reddit定番スレ。日本語圏と英語圏の両方。
【各候補について必ず】①何が最高峰か ②盗むべき機能/UX ③捨てるべき部分
④ライセンス を出力。
【最後に結論】「そのまま使える最有力」と「参考にする場合に盗む機能リスト」。
出典URL必須。存在しないものを創作しないこと。
```

## 調査依頼文② ObsidianとAIを組み合わせる運用の最高峰

```
あなたは「世界の最高峰事例を調べ尽くす調査員」です。
【需要】ObsidianのフォルダをClaude CodeなどのAIに直接読み書きさせる運用の、
2026年時点の最高峰事例・定番プラグイン・フォルダ構成を知りたい。
【調査範囲】GitHub、Zenn/Qiita/note等の日本語記事、英語圏のブログ・フォーラム。
【各候補について必ず】①何が優れているか ②自分の運用に取り入れるべき点
③不要な点 を出力。
【最後に結論】「明日から取り入れるべき上位3つ」。出典URL必須。
```

```
サーバー不要・オフラインで使えるものを優先。
【調査範囲】GitHub(スター順と直近1年更新)、Product Hunt、Hacker News。
【各候補について必ず】①何が最高峰か ②盗むべき機能/UX ③捨てるべき部分
④ライセンス を出力。
【最後に結論】「そのまま使える最有力」と「自作する場合のベースと盗む機能リスト」。
出典URL必須。存在しないものを創作しないこと。
```

## 新しい調査を頼みたくなったら(穴埋め文)

```
あなたは「世界の最高峰事例を調べ尽くす調査員」です。以下の需要を満たす既存の
ソフトウェア/OSS/商用製品/有名アプリを世界単位で調査してください。

【需要】(欲しいものを2〜3行で。何をする道具か、週に何回使うか)
【絶対条件】(譲れない条件。例: オフラインで動く、無料、日本語データ対応)
【調査範囲】GitHub(スター順と直近1年更新の両方)、商用製品、Hacker News、
Reddit定番スレ。日本語圏と英語圏の両方。
【各候補について必ず】①何が最高峰か ②盗むべき機能/UX ③捨てるべき部分
④絶対条件との適合 ⑤ライセンス を出力。
【最後に結論】「そのまま使える最有力」「自作する場合のベースと盗む機能リスト」。
出典URL必須。存在しないリポジトリを創作しないこと。
```

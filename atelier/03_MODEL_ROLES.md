# 03 MODEL_ROLES — モデル役割表と検収基準

最高性能モデルの時間は最も高価な資源。**「その作業は本当にFable5級が必要か?」を毎回問う**(憲法第7条)。モデル名は世代交代するため「級」で定義する。

## 役割表

| 級 | 例 (2026-07現在) | 担当 | 渡すもの |
|---|---|---|---|
| **蒸留級** | Claude Fable5 | インサイト確定の対話 / アーキテクチャ・情報設計 / 最難関実装(★印) / 最終検収 / 憲法改訂 | 生の欲望・長文の思考。汲み取りが仕事 |
| **量産級** | Codex / GPT系コーディング / Claude Sonnet | 指示書ベースの実装 / リファクタ / テスト追加 / M2以降の磨き込み | briefs/ の指示書1本(自己完結) |
| **調査級** | ChatGPT DeepResearch / Gemini | 世界単位の最高峰事例調査 | B02のプロンプトパック |
| **整形級** | Claude Haiku / 安価API | データ変換 / 表整形 / frontmatter付与 / 検証スクリプト実行 / 定型文書 | 入出力例つきの単純明快な指示 |

判断に迷ったら: **不可逆な判断(設計・取捨選択)は上へ、可逆な作業(実装・整形)は下へ。**

## 指示書(brief)の渡し方

1. `templates/brief.md` から作成し `briefs/Bxx_名前.md` に置く
2. 指示書は**自己完結**させる: 会話履歴なしで実行できること(背景・参照ファイル・参照最高峰・仕様・検収基準を全て内包)
3. 渡し先がリポジトリを読める場合はパスで参照、読めない場合(ChatGPT等)は指示書に必要コードを同梱
4. 1指示書 = 1ブランチ = 1成果物。混ぜない

## 検収基準 — 完了の定義 (憲法第8条)

以下の証跡が揃って初めて「完了」。揃っていない報告は差し戻す。

- [ ] **動作証跡**: 実際に動かした結果(テスト出力・スクリーンショット・実行ログ)。「動くはず」は不可
- [ ] **ビルド成立**: 成果物が生成コマンド一発で再現できる
- [ ] **オフライン維持**: (Webツールの場合) 実行時ネットワーク0件・localStorage不使用の確認
- [ ] **格好良さ**: スクリーンショットを見てテンションが上がるか(第4条)。上がらなければ差し戻し
- [ ] **ドキュメント**: SPEC/READMEが実装と一致
- [ ] **台帳更新**: [02_MASTERPLAN](02_MASTERPLAN.md) の状態と台帳を更新

> 実例: M0は「node --checkが通った」だけで完成報告されていたが、npm installすら失敗していた。動作証跡の無い完了報告を信用しないこと。

## ブランチ・リポジトリ運用

- main は常に検収済みのみ。作業はブランチ → 検収 → マージ
- コミットメッセージは日本語可。何を検証したかを本文に書く
- 生成物(dist/, node_modules/, testdata/)はコミットしない

# B04 AIファイル連動 — 分析バンドル書き出し

> リポジトリ keisuku/biz001。M1マージ済みが前提。着手前に `atelier/00_DOCTRINE.md` 第5条と `docs/SPEC.md` を読むこと。

## 背景

憲法第5条: AI連動はAPIでなくファイル境界で行う。Pivot Benchで集計している最中に「この未分類をどう分類すべきか」「この傾向をどう解釈するか」をAIに相談したくなる。そのときAPIを組み込むのではなく、**現在の分析状態を1つのMarkdownに書き出し、チャットAI(Claude/ChatGPT)に貼るだけ**で文脈が完全に伝わるようにする。

## 参照する既存資産

- `src/app.js` — `download()` / ビュー設定JSON v2 / `applyMaster()` の未分類統計 / Perspective `view.to_csv()`
- 出力形式の思想: `atelier/templates/` のfrontmatter規約(summary付きMarkdown)

## 仕様

1. ヘッダーに「AI相談バンドル」ボタンを追加。クリックで `pivot-bench-bundle-YYYYMMDD-HHmm.md` をダウンロード+同内容をクリップボードへコピー
2. バンドルの内容(上から順に):
   - frontmatter: summary(ファイル名・行数・マスタ名)、生成日時
   - **データ概要**: レポート名(複数なら全て)、行数、列一覧と型(number/string)、結合モードか否か
   - **適用マスタ**: マスタJSON全文(コードブロック)
   - **未分類レポート**: 未分類件数、未分類の対象列値の上位20件(値と件数)。これがAI相談の主目的(「これらに合うルールを提案して」)
   - **現在のビュー**: アクティブペインのビュー設定JSON+集計結果の先頭50行をCSVコードブロックで
   - **相談テンプレ**: 末尾に「上記を踏まえて: 」と定型の質問文3種(ルール提案/傾向解釈/次に見るべき切り口)をコメントとして同梱
3. 個人情報への配慮: 生データ全行は含めない(概要・上位値・集計結果のみ)。バンドル生成時にトーストで「集計値と上位値のみ含まれます」と表示
4. 逆方向: AIが返したルール提案(マスタJSON)は既存の「マスタを開く」で取り込める。バンドルの相談テンプレに「回答はマスタJSON全文で」と書いておくことで往復が閉じる

## やらないこと

- APIキーを使う直接連携、ストリーミング、チャットUI内蔵

## 検収基準

- [ ] 50,000行テストデータでバンドル生成→内容が仕様の全セクションを含む(ファイル添付)
- [ ] 生成したバンドルを実際にチャットAIへ貼り、文脈説明ゼロでルール提案が返ること(往復ログ添付)
- [ ] 返ってきたマスタJSONが「マスタを開く」でそのまま取り込めること
- [ ] 既存テスト全パス+実行時ネットワーク0件維持

# B02 世界最高峰リサーチ網

> ChatGPT DeepResearch / Gemini 等、Web調査能力のあるモデルにそのまま貼って使う。
> 成果物は `atelier/templates/research.md` の形式で受け取り、`atelier/research/Rxx_*.md` として保存する。

## 背景

憲法第2条「ゼロから創らない」を支える目。新しい欲望が出るたびに世界単位で最高峰事例を調べ、「そのまま使えるか/何を盗むか」を判定する材料を作る。調査は安く外注できる代表格なので、テンプレ化して量産する。

## 調査プロンプトテンプレート(これを埋めて投げる)

```
あなたは「世界の最高峰事例を調べ尽くす調査員」です。以下の需要を満たす既存の
ソフトウェア/OSS/商用製品/有名アプリを世界単位で調査してください。

【需要】(インサイトファイルの①②を貼る)
【must要件】(箇条書き)
【制約】個人利用・オフライン動作を優先。ライセンス明記必須。

調査範囲: GitHub(スター順と直近1年更新の両方)、Product Hunt、Hacker News、
Reddit の定番スレッド、商用製品。日本語圏と英語圏の両方。

各候補について必ず: ①何が最高峰か ②盗むべき機能/UX ③捨てるべき部分
④must要件との適合表 ⑤ライセンス を出力。
最後に「そのまま使える最有力」「差分を作る場合のベースと盗む機能リスト」を結論として出力。
出典URL必須。存在しないリポジトリを創作しないこと。
```

## 初回発注(3本)

1. **R02 マスタ/ルールエンジン管理UI**: 「表形式のルールをブラウザで編集・並べ替え・検証するUIの最高峰」(P1 M2の参考)
2. **R03 Obsidian×AI運用**: 「Claude Code等がvaultを直接読み書きする運用の最高峰事例・プラグイン・フォルダ構成」(P2の継続改善)
3. **R04 個人ダッシュボード/ランチャー**: 「自分専用スタートページ・ランチャーのOSS最高峰(単一HTML/静的で動くもの)」(P6の材料)

## 検収基準

- [ ] 結論(最有力/ベース/盗むリスト)が長文でなく1画面で読める分量

# B05 私生活インテーク — 発見フェーズ

> これはコードを書く指示書ではない。**作る前に、何が本当に欲しいかを確定させる面談の台本**である(憲法第1条)。チャットAI(Fable5級推奨)にこのファイルを貼り、面談を実施する。

## 背景

「人生全域を自分専用ツールで固める」計画のうち、私生活領域(家計・健康・情報収集・その他)は要望が未言語化。仕事領域(Pivot Bench)はインサイトが明確だったから成功した。同じ精度の言語化を私生活でも行ってから、調査→紹介→カスタマイズに進む。

## 面談の台本(AIへの指示)

あなたは需要の取捨選択を助ける対話者である。以下を1テーマずつ、合計30分程度で聞き出す。
**解決策を提案しない。既存アプリの名前を出さない。欲望の言語化だけに徹する。**

1. **棚卸し**: 「直近1週間で、面倒だった・イライラした・後回しにした私生活の作業を、思いつくまま全部挙げてください」(最低10個出るまで待つ)
2. **頻度と痛み**: 各項目に週何回×何分かかるかを付ける。痛み上位3つを選ばせる
3. **深掘り(上位3つのみ)**: insight.md の①の項目(動詞/頻度/場面/現状/成功の絵)を全て埋める
4. **切る**: 「上位3つのうち、実は無くても困らないものはどれか」を問い、最低1つ切らせる(取捨選択の練習)
5. **格好良さ**: 残ったテーマについて「どんな見た目・手触りならテンションが上がるか」を聞く(参考にしている好きなアプリ・ガジェットを挙げてもらう)

出力: テーマごとに `templates/insight.md` 形式のファイル(status: インサイト確定)。

## 面談後の手順

1. 確定したインサイトを `10_Insights/`(vault)または `atelier/insights/`(リポジトリ)へ保存
2. [B02_リサーチ網](B02_リサーチ網.md) のテンプレで調査を発注(1テーマ1本)
3. 調査結果の「紹介」判定は蒸留級と行う。**既存アプリで足りるならそれで終了が最良**
4. 差分を作る場合のみ、brief化して量産級へ

## 検収基準

- [ ] insight.md 形式で1〜2テーマが「インサイト確定」status になっている(全項目埋まり)
- [ ] 最低1つのテーマが意図的に「見送り」になっている(切る練習の証跡)
- [ ] 各テーマに格好良さ要件が具体的に書かれている(「シンプルで使いやすい」のような曖昧語は差戻し)

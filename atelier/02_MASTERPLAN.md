# 02 MASTERPLAN — ¥10億Fable5計画

前提: 「最高峰モデルを残り数日間、無制限に使えるなら自分のために何をやるか」を列挙し、実際には他モデルとの役割分担で後日実現していく。**Fable5級は各プロジェクトの★印(最難関)のみ**を担当する。

## プロジェクト一覧

### P0 アトリエ — この体制そのもの 【状態: 稼働開始 2026-07-04】
憲法・手順書・指示書テンプレ・モデル役割表。**仕組み自体が最大の資産**(全プロジェクトの生産性に掛け算で効く)。
★Fable5: 憲法とパイプラインの蒸留(本日完了) / 以後の運用・微修正: どのモデルでも可

### P1 Pivot Bench — 仕事: Salesforce分析マシン 【状態: M0完了・M1本日】
SalesforceレポートCSV/xlsxのオフライン正規化+バケットマスタ分類+3面ピボット比較。単一HTML。
- M0 土台(完了): 正規化・マスタ・Perspective 3面・JSON入出力
- **M1 レポート上位互換(★Fable5・本日)**: プリセットバー(行キー最大4×列分割1×集計)、複数レポート並置/結合
- M2 磨き(Codex級): マスタ編集UI・保存プリセット・結果書き出し → [briefs/B01_pivot-bench-M2.md](briefs/B01_pivot-bench-M2.md)
- M3 AI連動(Sonnet級): 分析バンドル書き出し → [briefs/B04_AIファイル連動.md](briefs/B04_AIファイル連動.md)

### P2 インサイト工房 — 知的生産: Obsidian vault 【状態: 指示書済】
PIPELINE を回す運用場。Inbox→Insights→Research→Projects の Markdown 資産。AI(Claude Code等)が vault を直接読み書きする(第5条)。
★Fable5: 構造設計(指示書内で完了) / 構築作業: 安価モデルで可 → [briefs/B03_インサイト工房vault.md](briefs/B03_インサイト工房vault.md)

### P3 世界最高峰リサーチ網 — 温故知新の目 【状態: 指示書済】
領域別「最高峰カタログ」を DeepResearch 系モデルに常時委託する仕組み。プロンプトパック+成果物フォーマット統一。
★Fable5: 不要(テンプレ設計のみ、済) → [briefs/B02_リサーチ網.md](briefs/B02_リサーチ網.md)

### P4 AI工場 — モデル役割分担基盤 【状態: 稼働開始】
[03_MODEL_ROLES](03_MODEL_ROLES.md) + briefs テンプレの運用。指示書の書き方・検収基準・ブランチ運用を固定化し、どのモデルに渡しても同品質にする。
★Fable5: 役割表と検収基準の設計(本日完了) / 運用: 人間+任意モデル

### P5 私生活ツール群 — 人生全域への展開 【状態: 発見フェーズ待ち】
家計・健康・情報収集など。**要望が未確定のため、作る前にインサイト面談(発見フェーズ)が先**(第1条)。
★Fable5: 面談の対話相手とインサイト蒸留 → [briefs/B05_私生活インテーク.md](briefs/B05_私生活インテーク.md)
面談で確定したものから P1 と同じ流れ(調査→紹介→カスタマイズ)に投入。

### P6 パーソナルホーム — 全ツールの入口 【状態: 構想】
自分専用ランチャー(オフライン単一HTML)。ツール起動・インサイトInbox・進行中プロジェクト表示。P1/P2 が育ってから着手(入口だけ先に作ると空箱になる)。
★Fable5: 情報設計 / 実装: Codex級

### P7 業務ドメイン資産 — 保険代理店ナレッジの蓄積 【状態: 構想】
正規化ルール・商材バケットマスタ・列名辞書を Pivot Bench から独立した JSON/Markdown 資産に切り出し、vault で版管理。ツールが変わっても資産は残る。
★Fable5: 不要 / 抽出・整理: Haiku級で可

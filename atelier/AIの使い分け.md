# AIの使い分け — 早見表と合格チェック

最高性能のAIの時間は一番高い資源。**「その作業は本当に最高性能が必要か?」を毎回問う**(憲法 第7条)。AIの名前は世代交代するので「級」で覚える。

## 早見表

| 級 | 例 (2026-07現在) | 頼むこと | 渡すもの |
|---|---|---|---|
| **蒸留級** | Claude Fable5 | 欲しいものの聞き出し・全体の設計・一番難しい実装・最終の合格判定・憲法の改訂 | 生の欲望・長文の思考。汲み取りが仕事 |
| **量産級** | Codex / GPT系 / Claude Sonnet | お願い書どおりの実装・改修・テスト追加・磨き込み | [やること/](やること/) のお願い書1本 |
| **調査級** | ChatGPT Deep Research / Gemini | 世界の最高峰事例の調査 | [やること/2_ChatGPTに貼る_世界の最高峰ツールを調べてもらう.md](やること/2_ChatGPTに貼る_世界の最高峰ツールを調べてもらう.md) の依頼文 |
| **整形級** | Claude Haiku / 安いAPI | データ変換・表の整形・定型文書・テストの実行 | 入出力の例つきの単純明快な指示 |

迷ったら: **やり直しがきかない判断(設計・いる・いらない)は上へ、やり直せる作業(実装・整形)は下へ。**

## お願い書の渡し方

1. [ひな形/お願い書.md](ひな形/お願い書.md) から作って [やること/](やること/) に置く。ファイル名は「番号_誰に渡す_何をしてもらう.md」
2. お願い書は**それ1枚で完結**させる: 会話の履歴なしで実行できること(背景・参考にする最高峰・作るもの・合格チェックを全部書く)
3. 渡し先がこのリポジトリを読める場合(Codex/Claude Code)はファイル名を伝えるだけ。読めない場合(ChatGPT等)は貼るだけで済む形にする
4. お願い書1本 = 成果物1つ。混ぜない

## 合格チェック — 「完了」の定義 (憲法 第8条)

以下が揃って初めて完了。揃っていない完成報告は「証拠を見せて」と差し戻す。

- [ ] **動いた証拠**: テスト結果の出力・スクリーンショット・実行ログ。「動くはず」は不可
- [ ] **作り直せる**: 成果物がコマンド一発で再現できる
- [ ] **オフライン維持**(Webツールの場合): 実行時ネットワーク0件・localStorage不使用
- [ ] **格好良さ**: スクリーンショットを見てテンションが上がるか。上がらなければ差し戻し
- [ ] **文書が一致**: 仕様書・READMEが実装と合っている
- [ ] **記録**: [全体計画.md](全体計画.md) の記録表に1行追記されている

> 実例: Pivot Benchの土台は「文法チェックが通った」だけで完成報告されていたが、実際は部品の取得すら失敗していた。動いた証拠の無い完了報告を信用しないこと。

## リポジトリの決まりごと

- main には合格したものだけ。作業はブランチ → 合格 → 取り込み
- コミットメッセージには「何を検証したか」を書く
- 生成物(dist/、node_modules/、testdata/)はコミットしない

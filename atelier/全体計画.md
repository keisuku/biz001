# 全体計画 — これから作るもの一覧

前提: 「最高性能のAIを使い放題なら自分のために何をやるか」を全部書き出し、実際にはAIの使い分け([AIの使い分け.md](AIの使い分け.md))で少しずつ実現していく。**難しい所だけ最高性能のAI(Claude Fable5級)にやらせ、残りは安いAIで回す。**

## 作るもの一覧

### このアトリエ自体 【動き始めた 2026-07-04】
憲法・手順・お願い書のひな形・AIの使い分け表。**この仕組み自体が一番の資産**(この先作る全部の道具の生産性に効く)。

### Pivot Bench — 仕事: Salesforce分析マシン 【土台と本体は完成。次は磨き込み】
SalesforceレポートのCSV/xlsxをオフラインで正規化し、商材3階層に自動分類して、3画面でピボット比較する単一HTMLツール。
- 土台(完成): 正規化・分類マスタ・3面表示・JSON入出力
- 本体(完成): プリセットバー(行4×列分割×集計)、レポート3本の並置/結合
- 磨き込み(次): マスタ編集画面・お気に入り集計・結果書き出し → [やること/3_Codexに渡す_PivotBenchの磨き込み.md](やること/3_Codexに渡す_PivotBenchの磨き込み.md)
- AI相談機能(その次): → [やること/4_Codexに渡す_PivotBenchにAI相談機能を足す.md](やること/4_Codexに渡す_PivotBenchにAI相談機能を足す.md)

### 知識の保管庫 — 知的生産: Obsidian 【お願い書は用意済み。あとは貼るだけ】
アイデア・調査・お願い書を貯めて、AIに直接読み書きさせる保管庫。
→ [やること/1_ChatGPTに貼る_Obsidianに知識の保管庫を作ってもらう.md](やること/1_ChatGPTに貼る_Obsidianに知識の保管庫を作ってもらう.md)

### 世界の最高峰を調べる仕組み 【お願い書は用意済み。必要になったら貼るだけ】
「ゼロから作らない」を支える目。調査依頼文が定型化してあるので、コピペで何度でも使える。
→ [やること/2_ChatGPTに貼る_世界の最高峰ツールを調べてもらう.md](やること/2_ChatGPTに貼る_世界の最高峰ツールを調べてもらう.md)

### WEBマーケ診断と10年ロードマップ — 仕事: 丸紅セーフネット 【お願い書は用意済み。新しいセッションで走らせるだけ】
LP(marubeni-safenet.com)とコーポレートサイト(m-inc.co.jp)のSEO・導線・競合を外部診断し、甘い部分の指摘と10年ロードマップを作る。GA4分析ツールのKPI設計の土台になる。
→ [やること/6_新しいセッションで_丸紅セーフネットのWEBマーケ診断と10年ロードマップ.md](やること/6_新しいセッションで_丸紅セーフネットのWEBマーケ診断と10年ロードマップ.md)

### GA4分析ツール — 仕事: データドリブンな経過観察 【まず最高峰調査から】
GA4のエクスポートを入れると分析資料(月次・傾向・異常・LP別成績)が出る自分専用ツール。オフライン単一HTML想定。**実装は「診断でKPIが決まる+調査が返ってくる」の両方が揃ってから。**
→ [やること/7_ChatGPTに貼る_GA4分析ツールの最高峰を調べてもらう.md](やること/7_ChatGPTに貼る_GA4分析ツールの最高峰を調べてもらう.md)

### 社内DX総点検 — 仕事: システムの棚卸し 【まず面談から】
Salesforce・BOX・カオナビ・バクラク・Excel集計・JavaScript開発資産・ワインセラー管理の台帳づくりと痛みランキング。上位テーマだけを調査→紹介→自作の流れに乗せる。
→ [やること/8_Claudeと話す_会社のDXと社内システムの棚卸し面談.md](やること/8_Claudeと話す_会社のDXと社内システムの棚卸し面談.md)

### 私生活の道具たち 【まず聞き出し面談から】
家計・健康・情報収集など。何が欲しいかがまだ言葉になっていないので、**作る前に面談で聞き出す**。
→ [やること/5_Claudeと話す_私生活で欲しいものを聞き出してもらう.md](やること/5_Claudeと話す_私生活で欲しいものを聞き出してもらう.md)
面談ではっきりしたものから、調査→紹介→自作の流れ([作り方の手順.md](作り方の手順.md))に乗せる。

### 自分専用のスタートページ 【構想段階。急がない】
全部の道具の入口になる1画面(道具の起動・メモの入口・進行中の一覧)。オフライン単一HTML。他の道具が育ってから作る(入口だけ先に作ると空箱になる)。

### 仕事の知識の資産化 【構想段階。急がない】
分類マスタ・列名の辞書・正規化ルールを、道具から独立したJSON/文書として保管庫に貯める。道具を作り替えても知識は残る。

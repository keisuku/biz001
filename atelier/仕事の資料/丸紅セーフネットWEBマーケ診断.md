# 丸紅セーフネット WEBマーケ診断と10年ロードマップ

**診断の前提(重要)**
- 本診断は外部からの調査のみで作成した。対象サイト2つ(m-inc.co.jp / marubeni-safenet.com)は自動アクセスに403を返すため、**ページの実物(デザイン・CTA配置・構造化データ・表示速度)は未確認**。検索エンジン経由で見える範囲(ページ一覧・title・スニペット・検索結果での出現)のみを事実として扱い、見えなかった部分は「持ち主に頼むこと」に回した。
- 検索結果の観察は米国経由の検索APIによるもの。**日本のGoogleでの実順位とは一致しない可能性がある**ため、「何位」とは断定せず「検索結果に出た/出なかった」の水準で記述する。実順位はSearch Consoleのデータで確定させる(持ち主に頼むことリスト参照)。
- 持ち主(あなた)からの前提: 商社系代理店は外売りをほぼしておらず**WEB上の直接競合は現状不在。丸紅セーフネットはこの領域でNo.1かつOnly.1**。よって本診断の軸は「競合に勝つ」ではなく、①独走ゆえに気づきにくい甘さの発見、②現代のB2Bマーケ世界標準との差分、③独走を10年守り広げる参入障壁づくり、の3つ。

## 結論サマリー(まずここだけ)

**現状の一言評価: 「商品LPの間口は強いが、家の土台(計測)と資産(コンテンツ)が無い」。**
「取引信用保険 比較 見積り」「D&O保険 比較 見積り」では検索結果の先頭に自社ページが出ており、ニッチ商品×比較・見積り意図の検索は現状ほぼ独占できている。一方で、記事型コンテンツはゼロ(検索から1本も見つからない)、同一商品のページが2ドメインに重複し、計測体制は外から確認できない。**感覚で当ててきた強い間口を、データと資産で固定化する段階**にある。

**一番効く打ち手3つ:**

1. **計測体制の確立(GA4+Search Console)を最優先で確認・整備する。** 何が当たっているかをデータで見られない限り、本診断を含む全ての施策の良し悪しが判定不能。これが「やること/7(GA4分析ツール)」の入口になる。
2. **2ドメインの役割重複を解消する。** 取引信用保険・D&O・サイバー・表明保証の解説ページが両ドメインに存在し(取引信用保険は3ページ)、検索評価が分散している。「m-inc=会社情報、marubeni-safenet=商品と見積り」に寄せ、重複ページは統合かcanonicalで一本化する。
3. **専門コラム(記事資産)を新設し、AI回答面に布石を打つ。** 現状は商品LPのみで、ChatGPT等のAIが法人保険を回答するときに引用できる情報源として存在していない。競合不在の今なら、法人保険の専門記事を積むだけで「AIが引用する唯一の代理店」になれる。これが10年スパンで最大の参入障壁になる。

## 指摘事項(甘い部分)

各項目は「事実 → なぜ問題か → 放置するとどうなるか」の3点セット。推測には(推測)と付けた。

### 指摘1: 同一商品の解説ページが2ドメインに重複している

- **事実**: 検索結果から、少なくとも次の重複を確認した。
  - 取引信用保険: `m-inc.co.jp/corporation/trade-credit-insurance/`、`m-inc.co.jp/business/torihiki.html`、`marubeni-safenet.com/products/products-trade-credit/` の**3ページ**
  - D&O保険: `m-inc.co.jp/corporation/d-and-o-insurance/` と `marubeni-safenet.com/products/products-do/`
  - サイバー保険: `m-inc.co.jp/corporation/cyber-insurance/` と `marubeni-safenet.com/products/products-cyber/`
  - 表明保証保険: `m-inc.co.jp/corporation/representation-and-warranty-insurance/` と `marubeni-safenet.com/products/rwi/`。「表明保証保険」の検索では**両ドメインが同じ検索結果ページに別々に出現**した
- **なぜ問題か**: 検索エンジンは同一テーマのページが分かれているとリンク・評価も分散させる。せっかくの強いテーマで、1本なら上位に固定できる力を2〜3本に割っている。またユーザーから見ても、どちらが「本物の商品ページ」か分からず導線が濁る
- **放置するとどうなるか**: 将来競合が1本の強いページで参入してきたとき、こちらは分散した2〜3本で迎え撃つことになり、独走中に築いたはずの検索資産が目減りする。ドメインをまたぐ重複は後から統合するほど(リダイレクト・リンク張り替えの)コストが増える

### 指摘2: 記事型コンテンツ(コラム・事例・お役立ち)が検索上に存在しない

- **事実**: `site:marubeni-safenet.com コラム OR お役立ち OR 事例` の検索で該当ページが1本も見つからなかった。m-inc.co.jp 側にも記事型ページは見つからない。一方、「取引信用保険」「サイバー保険」等の情報収集型の検索結果は、保険会社直販(東京海上日動・損保ジャパン)、公的機関(中小機構・損保協会)、比較メディア(インズウェブ・ライフィ・マネーフォワード)、士業・専門ブログで占められている
- **なぜ問題か**: 法人保険の購買は「事故や不安→情報収集→比較→問い合わせ」と進む。現状は最後の「比較・見積り」の検索しか取れておらず、その手前の大きな母集団(「取引信用保険とは」等で調べ始めた見込み客)を毎回メディアと保険会社に渡している。E-E-A-T(専門性・権威性)をGoogleに示す材料も商品LPだけでは薄い。さらにChatGPT等のAI回答は既存のWeb記事を情報源にするため、**記事が無い=AIの回答に登場できない**
- **放置するとどうなるか**: 検索の入口がAIチャットに移るほど、「比較・見積り」ページだけのサイトは指名されなくなる。10年後の顧客が「取引信用保険ってどうすればいい?」とAIに聞いたとき、答えの中に丸紅セーフネットが出てこない未来が確定する

### 指摘3: 商品によって検索での見え方に大きなムラがある

- **事実**(いずれも本診断時の検索結果への出現有無。実順位はSearch Consoleで要確定):
  - **出た(強い)**: 「取引信用保険 比較 見積り」→ 結果先頭に自社。「D&O保険 会社役員賠償責任保険 比較 見積り」→ 結果先頭に自社。「事業包括保険」「表明保証保険 M&A」「サイバー保険 法人 比較」→ 結果内に自社が出現
  - **出なかった(弱い)**: 「法人 保険 代理店 比較」「労災 上乗せ保険 法人 比較」→ 上位10件に自社なし。労災上乗せは自社ページのtitleが「政府労災保険の上乗せ補償」で、世間の呼び名「労災上乗せ保険」と言葉がずれている
- **なぜ問題か**: 強い商品(取引信用・D&O)は「商品名+比較・見積り」という**購買直前のニッチ検索**で勝っている。これは正しい戦い方だが、どの商品がなぜ勝てていて、どこに検索需要が残っているかが体系的に管理されていない(推測: titleの付け方のばらつき—「保険会社44社から」が入るページと入らないページがある—から、ページごとに個別最適で作られてきたと見られる)
- **放置するとどうなるか**: 勝てるはずのキーワード(労災上乗せ等)を長期間取りこぼす。また「感覚と古い知識」で当ててきた成功パターンが言語化されないままだと、担当が変わった瞬間に再現できなくなる

### 指摘4: 活動実績が自社ドメインに資産化されていない

- **事実**: フェムテックサービス導入(ルナルナ オフィス)、サイバーセキュリティ対策セミナー共催、ファミワンとの提携などの活動が、丸紅本体サイト(marubeni.com)のニュースやPR TIMESには載っているが、自社2ドメインのニュース・実績ページは検索から見つからない。ブランド検索「丸紅セーフネット」の結果も、公式トップ3件の下は採用口コミサイト(OpenWork・転職会議・エン)が占める
- **なぜ問題か**: 活動はしているのに、その信頼の証拠が自社ドメインに蓄積されず、他社ドメイン(丸紅本体・PR TIMES)の資産になっている。E-E-A-Tの「実績」を自社サイトで示せないのはもったいない。ブランド検索の2位以下が口コミサイトなのは、法人顧客が社名で調べたときの見え方として管理されていない状態
- **放置するとどうなるか**: 10年分の活動実績が全部よそのドメインに積み上がり、自社サイトは「昔作った商品カタログ」のまま古びていく。見込み客・採用候補者が社名検索したときの印象を口コミサイトに委ね続けることになる

### 指摘5: 自動アクセスの403ブロックがAIクローラーも弾いている可能性

- **事実**: 両ドメインとも本診断環境からの機械アクセスに403を返した(確認済み)。どのbot・どの経路を弾いているかの設定内容は未確認
- **なぜ問題か**(一部推測): セキュリティとしては真っ当だが、設定が広すぎるとGooglebotは通してもAI検索(ChatGPT・Perplexity・Gemini等)のクローラーを弾いている可能性がある。指摘2のとおりAI回答面はこれからの主戦場であり、**存在するのに読めないサイトは、AIにとって存在しないサイト**になる
- **放置するとどうなるか**: 記事資産をこれから作っても、クローラー設定次第ではAIの学習・引用対象に入らず、投資が無駄になる。robots.txtとWAFの設定確認は先にやるべき(持ち主リスト参照)

### 指摘6: 基本数値の表記が揺れている疑い(要確認)

- **事実**: 検索スニペット上で、取引先数が「約3,000社」(m-inc.co.jp企業情報)と「約2,500社」(m-inc.co.jp英語ページ)の2つの数字で出現した。**スニペットは古いキャッシュの可能性があるため断定しない**
- **なぜ問題か**: もし実際に揺れているなら、ページごとに手で更新してきた(=更新が属人的で棚卸しされていない)兆候。数字の不一致は法人顧客・保険会社・監査的な目線で信頼を削る
- **放置するとどうなるか**: サイトが大きくなるほど不整合が増え、全ページ棚卸しのコストが膨らむ。会社の基本情報は1か所で管理して全ページに反映される作りにしておくべき

## 改善の方向性(指摘に対応)

### すぐやる(〜1か月)
1. **計測の現状確認**: GA4・Search Console・GTMの導入有無と設定を棚卸しし、問い合わせ送信を「キーイベント」として計測できているか確認する(→指摘全部の判定基盤)
2. **robots.txt・WAF設定の確認**: Googlebot・Bingbot・主要AIクローラー(GPTBot, ClaudeBot, PerplexityBot等)の許可状況を確認し、方針を決めて明示する(→指摘5)
3. **重複ページの一覧化**: 2ドメインの全ページを棚卸しし、商品ごとに「正となる1ページ」を決める(→指摘1)。すぐ消さず、まずどのページに検索流入が来ているかSearch Consoleで確認してから統合順を決める

### 半年
4. **ドメイン役割の一本化**: 「m-inc.co.jp=会社・採用・グループ向け」「marubeni-safenet.com=商品・見積り・記事」に整理。重複ページは301リダイレクトかcanonicalで正ページに集約(→指摘1)
5. **title/descriptionの規格化**: 全商品LPで「商品名+比較・見積り+44社+丸紅」の型に統一。「労災上乗せ保険」など世間の呼び名に合わせた言葉に直す(→指摘3)
6. **ニュース・実績セクションの新設**: 提携・セミナー・導入事例を自社ドメインに時系列で載せる。過去分(ルナルナ・ファミワン・セミナー等)もさかのぼって掲載(→指摘4)

### 1年
7. **専門コラムの立ち上げ**: 商品ごとに「〜とは」「事例」「よくある質問」の3本柱で、月2本ペースの持続可能な体制を作る。書き手は社内の商品担当+AI下書きで、実案件の知見(匿名化)を必ず1段落入れて差別化する(→指摘2)
8. **構造化データ(FAQ・Organization・BreadcrumbList)の実装**とAI回答面での出現モニタリング開始(→指摘2・5)
9. **成功パターンの言語化**: どのキーワード×どのページ×どんな問い合わせが売上につながったかをデータで振り返り、「勝ちの型」を文書化する(→指摘3。やること/7のGA4分析ツールがここで効く)

## 10年ロードマップ — No.1/Only.1を「感覚の独走」から「構造的な独走」へ

### 第1段階(〜2年): 基盤整備と計測体制 — 「見える化」
- やること: 上記「すぐやる+半年」の全部。GA4/Search Consoleの完全整備、ドメイン統合、title規格化、ニュース資産化、コラム開始
- **達成の判定条件**:
  - [ ] 問い合わせ件数・経路が月次でダッシュボード1枚で見える(GA4キーイベント+チャネル別)
  - [ ] 主要商品キーワードの実順位がSearch Consoleで月次追跡されている
  - [ ] 商品ページの重複がゼロ(1商品=正1ページ)
  - [ ] 記事が24本以上あり、記事経由の問い合わせが1件以上発生している

### 第2段階(〜5年): データドリブン運用とコンテンツ資産化 — 「感覚をデータで置き換える」
- やること: 記事100本規模の専門ライブラリ(商品×業種×事例のマトリクス)。問い合わせデータと検索データを突合して「儲かるキーワード」に投資集中。メールマガジン等で見込み客リストを自社資産化。AI回答(ChatGPT等)での自社言及を四半期ごとに定点観測
- **達成の判定条件**:
  - [ ] 施策の意思決定が「先月のデータでこうだったから」で説明できる(感覚での意思決定に説明が付く)
  - [ ] 情報収集型キーワード(「取引信用保険とは」等)で主要5商品中3商品以上、検索1ページ目に自社記事がある
  - [ ] WEB経由の新規問い合わせ件数が第1段階終了時の2倍
  - [ ] 主要AIチャットに法人保険の相談をすると丸紅セーフネットが選択肢として言及される(定点観測で確認)

### 第3段階(〜10年): 検索とAI回答面での第一想起の維持と参入障壁化 — 「後発が追えない構造を作る」
- やること: 10年分の記事・事例・実績データという「時間でしか作れない資産」を武器に、検索でもAI回答でも「法人保険の相談=丸紅セーフネット」の第一想起を固定。実案件から得た統計(業種別リスク傾向等)の匿名化レポートなど、独自データの発信で唯一の一次情報源になる。後発が数億円かけても10年分の実績と一次データは買えない
- **達成の判定条件**:
  - [ ] 指名検索(丸紅セーフネット)の検索回数が10年前比で明確に増加トレンド
  - [ ] 主要商品の情報収集型・比較型キーワードの大半で1ページ目を維持
  - [ ] AI回答面での言及率が定点観測で安定的に維持されている(競合の出現を含めて監視)
  - [ ] WEB経由の問い合わせが営業の主要な新規源のひとつとして社内で認知されている(件数・成約データで裏付け)

## 付録: サイトを直接閲覧できるAIに渡す検証プロンプト

この診断環境からはサイト実物が見えなかったため、Web閲覧ができるAI(claude.aiのチャットでWeb検索オン、ChatGPT等)に下記を貼ると、未確認部分の実地検証ができます。返ってきた結果はこのファイルの下に追記してください。

```
あなたはWEBマーケティングの検証員です。丸紅セーフネット株式会社の2つのサイトを実際に閲覧して、以下を事実ベースで報告してください。アクセスできなかったページは「アクセス不可」と明記し、推測で埋めないでください。

対象:
- https://www.m-inc.co.jp (コーポレートサイト)
- https://marubeni-safenet.com (法人保険のマーケティングサイト)
```

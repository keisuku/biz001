# 作り方の手順 — 欲しいと思ってから完成まで

「はっきりさせる → いる・いらないを選ぶ → 世界の最高峰を調べる → 紹介 → 良いとこ取りを決める → 作らせる」を、誰(どのAI)がやっても同じ品質で回せるようにした手順。1つの欲望 = 1つのメモファイル。途中で止まっても、メモを見れば次のステップが分かる。

```
[思いつき] → ① はっきりさせる → ② いる・いらない → ③ 世界の最高峰を調べる
                                                        ↓
        ⑥ 作らせる ← ⑤ 良いとこ取りを決める ← ④ 紹介 (既存で足りればここで終了=一番の成功)
```

## ステップ0: 思いつき(いつでも・数秒)

「〜が欲しい気がする」「〜が面倒」と思った瞬間に [ひな形/欲しいものメモ.md](ひな形/欲しいものメモ.md) を複製して1行書く。整理不要。置き場所は保管庫の「00_思いつき/」(または本リポジトリ `atelier/` 内)。

## ① はっきりさせる (Claudeとの対話がおすすめ)

チェックリスト — 全部埋まるまで次に進まない:
- [ ] **何をする道具か**(動詞で。例: 「集計して並べて比較する」)
- [ ] **週に何回使うか**(週1未満なら作らない選択も検討)
- [ ] **いつ・どこで・何のデータで使うか**
- [ ] **今は何でどうやっていて、何分かかっているか**
- [ ] **完成した道具を使っている自分の情景を3行で**

## ② いる・いらないを選ぶ

機能の候補を **絶対いる / あったら嬉しい / いらない** に分ける。判断基準:
- 週1回以上使う操作か → 絶対いる
- 「便利そう」だけで使う場面が言えない → **いらない**(ここで切る勇気が品質)
- 見た目・手触りの希望(テンションが上がる条件)も「絶対いる」に書く(憲法 第4条)

## ③ 世界の最高峰を調べる (AIに任せられる)

世界単位で調べ尽くす。[やること/2_ChatGPTに貼る_世界の最高峰ツールを調べてもらう.md](やること/2_ChatGPTに貼る_世界の最高峰ツールを調べてもらう.md) の穴埋め文をコピペで発注し、結果は [ひな形/調査結果.md](ひな形/調査結果.md) の形式で [調査結果/](調査結果/) に保存する。

## ④ 紹介

調査結果から「既存のものをそのまま使えば終わりか?」を判定する。
- **足りる** → 導入して終了。作らずに済むのが一番の成功(憲法 第2条)
- **足りない** → 何が足りないか(差分)を1段落で言葉にする。その差分こそが作るものの定義

## ⑤ 良いとこ取りを決める

**引き算の設計**: 最高峰事例のどの機能・見た目を盗み、どれを捨てるか列挙する。
土台にする既存のもの(OSSや手持ちの道具)を1つ決める。自作するのは差分だけ。

## ⑥ 作らせる

- [ひな形/お願い書.md](ひな形/お願い書.md) でお願い書を書き、[やること/](やること/) に置く
- [AIの使い分け.md](AIの使い分け.md) を見てどのAIに渡すか決める
- 実装 → **合格チェック(憲法 第8条: 動いた証拠が必須)** → [全体計画.md](全体計画.md) の記録表に1行追記
